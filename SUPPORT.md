# Support

## Getting help

For usage and setup help:
- read `README.md`
- open `docs/README.md`
- review `docs/tools/TOOLS_MAP.md`

## Issue reporting

Open a GitHub issue with:
- expected behavior
- actual behavior
- reproduction steps
- logs or screenshots when available

## Feature requests

Use GitHub issues and include:
- problem statement
- proposed solution
- impact and trade-offs

## Security issues

For vulnerabilities, follow `SECURITY.md` and use private reporting.
